Hot glue 3D printer

This was my final project submission for my first year mechatronics engineering course. The printer extrudes fused hot glue sticks as filament.

First, an object is designed in solidworks and is exported as an .stl file. This file is then run through the Slic3r G-Code generator and the resulting gcode file is loaded onto an NXT brick. From here my software takes over, parsing the individual lines of G-Code to create a list of commands that the printer will execute. Commands such as "G1 X90.6 Y13.8 E22.4" tell the printer to move linearly to a specific point on the XY gantry. To move the priniter head, feedback from two encoders in the gantry are used as inputs to a PID loop that continually runs. This keeps the nozzle in the correct position at all times.

The printer is fairly versatile and can print many oddly shaped parts; however, due to the memory limitations of the NXT, only relatively simple parts can be constructed.

FIRST Robotics 2014

In 2014 I was a student member and lead programmer of the competitive FIRST robotics team called Simbotics. During my time in robotics I developed software that allowed our robot to be controlled visually with a Kinect camera during the "autonomous period" of each match. As seen in the video, the user's hand height is mapped to certain locations on the field.

The robot is constantly changing its set point based on the input from the Kinect. Using encoders as an input device, a PID loop is constructed where the set point is continuously changing. Since the robot is also very top heavy and rapidly changes direction, I also added auto stabilization by using the data about the changing set point and the robot's current speed.

Minesweeping Quadcopter

In the winter of 2014, I was enrolled into an engineering co-op program at Brock University where I worked alongside a professional engineer on a mechatronics project. I chose to create a minesweeping quadcopter from prebought and homemade components such as the custom PIC32 circuit board and ultrasonic sensor mounts. The custom circuit board that I designed is meant to facilitate the integration of a variety of sensors such as a metal detector and a camera. It processes all of this data and sends it out to a commercial quadcopter speed controller board.

I created a windows application that allows a user to send remote commands over an RF module to the quadcopter. This also allowed us to enable/disable the robot and receive its position and orientation data safely and efficiently.

FIRST Robotics 2015

As a mentor on my robotics team, I teach high school students programming methodologies for robotics applications, and developed some software as well.

The purpose of the robot this year was to stack industrial sized totes on a raised platform to score more points than the opposing robots. This meant that the robot needed to quickly traverse the field amid obstacles. To get around this problem we programmed an omnidirectional drivetrain to traverse the field in any direction without the need to turn.

Nerf Gun

This was an entry to the 2015 Enghack hackathon. It is a myo controlled nerf gun which mimics the user's movements and shoots on command via a fist gesture.

It is run by an arduino, which communicates to the myo via bluetooth to gather data about the orientation and current gesture of the hand. The arduino then sets the position of two servos which control the vertical and horizontal angles of the gun.

Fuel Cell Car

This is a culminating project for the mechatronics and chemistry courses. A car is to be powered from fuel cells in order to run two motors so that the car can traverse a maze on a tabletop. I created a line following algorithm in C++ that tracked the boundary of black electrical tape against the white background instead of just white or black. The tape is laid out across the entire table, so this algorithm allowed for extremely fast navigation throughout the maze. I also impemented a noise-cancelling filter for the button inputs on the car.

VEX Robotics

From 2011 to 2014 I served as the lead programmer for my VEX robotics competitive robotics team. Each match is composed of an autonomous period and a teleoperated period; I programmed various autonomous routines and the controls for the driver of the robot.

I worked with a variety of sensors such as a gyroscope, encoders, light sensors, potentiometers, line sensors and ultrasonics to develop reactive closed-loop algorithms for positioning and having precise control. I programmed all of these using C within the RobotC IDE.

In 2011, four other team members and I won the VEX Robotics World Championship in Orlando, Florida.

Function Grapher

This is a web application that makes use of fabric.js that I am currently developing. It allows the user to graph any function of their pleasing onto a grid. The entire graph moves with a click-and-drag motion and zooms as one would expect. I hope to improve the plotter by adding reactive axis numbers, and to display the function in latex form

Currently the plotter can graph any function relatively quickly, however functions with large slopes, or fast oscillations do not appear very smooth. I am trying to implement an algorithm that uses the derivative of the function to find points of local maxima and minima that appear on the graph at a certain zoom level, and use these as bounding points for the renderer. This will allow sharp functions like sin(1/x) to appear smooth and contained well within its proper range of (-1,1).
